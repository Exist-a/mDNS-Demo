# AGENTS.md

本文件定义本项目中 AI Agent 与协作者需要遵守的协作规范。

## 1. 测试驱动开发 (TDD)

所有功能开发与缺陷修复必须遵循测试驱动开发流程：

- **先写测试**：在实现任何新功能或修复之前，先编写对应的单元测试 / 集成测试，明确期望行为。
- **再写实现**：编写最小可用实现使测试通过。
- **后重构**：在测试通过的前提下优化代码结构,保持测试持续通过。
- 测试覆盖率应随功能同步提升,不得在未补充测试的情况下引入新逻辑。
- 提交前必须运行全部测试,确保无回归。

## 2. 修改后连带着修改文档

任何代码变更都必须同步更新相关文档,保持代码与文档的一致性：

- **代码变更** -> 检查并更新对应的:
  - `README.md` / `AGENTS.md` 中的使用说明、规范条目
  - 代码内 / API 注释、文档注释(GoDoc、JSdoc 等)
  - 配置文件示例、参数说明
  - `docs/` 目录下相关的设计文档、架构说明
- **文档变更** -> 同步核对代码是否仍与文档描述一致,必要时调整实现。
- 删除 / 弃用功能时,必须同时清理相关文档条目,避免遗留误导信息。

## 3. 解决不了的上网搜索或提问

遇到无法独立解决的问题时,按以下顺序处理:

- **先自查**:阅读相关源码、官方文档、注释,尝试理清上下文。
- **再搜索**:使用搜索引擎(优先官方文档、Stack Overflow、GitHub Issues)检索,关键词需包含语言 / 框架 / 库 / 错误信息。
- **再提问**:仍无法解决时,向用户提问,需提供:
  - 已尝试的方案与失败原因
  - 相关的错误日志、堆栈、复现步骤
  - 具体的疑问点与期望的答案
- 禁止在反复猜测中浪费时间,及时求助。

## 4. 没有明确说明的部分需要提问

对于需求、方案、实现细节中**没有明确说明**的部分,不得自作主张:

- 在动手实现前,主动使用 `AskUserQuestion` 等方式向用户确认。
- 需要确认的典型场景包括(但不限于):
  - 需求边界与异常处理策略
  - 第三方库 / 框架的选型
  - 接口签名、命名规范、目录结构
  - 向后兼容性与版本策略
- 宁可多问一句,也不要做出与用户预期不符的实现。
